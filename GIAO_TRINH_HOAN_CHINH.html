LỜI MỞ ĐẦU

Câu chuyện của một nhân viên bình thường

Hãy tưởng tượng bạn là Minh — trưởng nhóm hành chính của một công ty 200 người. Mỗi sáng thứ Hai, Minh mất gần 3 tiếng đồng hồ chỉ để tổng hợp báo cáo tuần từ 8 phòng ban khác nhau, copy-paste vào Excel, format lại và gửi email cho sếp. Công việc nhàm chán, lặp đi lặp lại, và cảm giác mình đang lãng phí thời gian làm những thứ không ai nhớ ơn.

Rồi một ngày, Minh phát hiện ra Antigravity.

Ba tuần sau, công việc báo cáo đó chỉ còn mất 7 phút. Minh đã tạo một ứng dụng web nhỏ tự động tổng hợp dữ liệu, vẽ biểu đồ, và xuất ra file PDF chuyên nghiệp. Không viết một dòng code nào. Không học lập trình. Chỉ cần nói chuyện với AI bằng tiếng Việt.

Minh dùng 2,5 tiếng còn lại đó để làm những việc thực sự quan trọng: gặp gỡ nhân viên, lên kế hoạch cải thiện quy trình, và — lần đầu tiên trong nhiều tháng — ra về đúng giờ.

Cuốn sách này được viết cho bạn — người đang ở vị trí của Minh ngày hôm đó.

Cuốn sách này dành cho ai?

Cuốn sách này được viết đặc biệt cho bạn nếu bạn:

  • Làm việc văn phòng: hành chính, kế toán, nhân sự, kinh doanh, marketing, vận hành...
  • Không có nền tảng lập trình — và không có ý định học
  • Cảm thấy mình đang mất quá nhiều thời gian cho các công việc lặp lại
  • Tò mò về AI nhưng chưa biết bắt đầu từ đâu
  • Muốn tạo ra công cụ/ứng dụng hỗ trợ công việc mà không cần nhờ IT

Bạn sẽ đạt được gì sau 30 ngày?

Tuần 1 — Nền tảng: Làm quen Antigravity, viết prompt hiệu quả, tiết kiệm 1-2 giờ/ngày ngay từ ngày đầu tiên.

Tuần 2 — Tăng tốc: Tự động hoá các tác vụ lặp lại, xây dựng thư viện công cụ cá nhân, đạt trạng thái làm việc x3 hiệu suất.

Tuần 3 — Vibe Coding: Tạo ứng dụng web thực sự từ ý tưởng — dashboard, form nhập liệu, công cụ phân tích — không cần code.

Tuần 4 — Làm chủ: Kết nối mọi thứ lại, xây dựng hệ thống tự động hoá hoàn chỉnh, đạt trạng thái x10 hiệu suất.

Cách đọc cuốn sách này

Mỗi ngày gồm 4 phần:
- 🧠 Bài học hôm nay — Khái niệm/kỹ năng cần nắm (đọc 5-10 phút)
- ⚡ Thực hành ngay — Bài tập cụ thể, làm luôn trong ngày (15-30 phút)
- 💡 Mẹo pro — Tip từ người dùng thực tế
- ✅ Checkpoint — Tự kiểm tra xem đã đạt chưa

Cam kết của bạn: 30 phút mỗi ngày, liên tục 30 ngày. Chỉ vậy thôi.

PHẦN 1: QUẢN LÝ WORKSPACE & THƯ MỤC

1.1 Workspace là gì?

Workspace = "Văn phòng ảo" nơi AI làm việc cùng bạn.
Mỗi dự án nên có 1 workspace riêng — giống như mở tab mới trên trình duyệt.

Ví dụ các workspace nên có:
✅ "bao-cao-kinh-doanh"     ← Workspace riêng cho báo cáo KD
✅ "quan-ly-cong-van"       ← Workspace riêng cho HC
✅ "du-an-crm-moi"          ← Workspace của từng dự án
✅ "hoc-tap-ca-nhan"        ← Workspace thực hành/thử nghiệm

❌ Dùng chung 1 workspace cho tất cả mọi thứ → AI bị nhầm ngữ cảnh

1.2 Tạo & Mở Workspace

Tạo workspace mới:

Cách 1: File → Open Folder → Chọn thư mục dự án
Cách 2: Kéo thả thư mục từ Finder/Explorer vào Antigravity
Cách 3: Gõ lệnh trong Terminal: [Antigravity tự mở]

Mở workspace đã có:

File → Open Recent → Chọn tên workspace từ danh sách
Hoặc: File → Open Folder → tìm đến thư mục

💡 Mẹo: Ghim workspace hay dùng vào "Favorites" (click phải → Add to Favorites) để mở nhanh hơn.

PHẦN 2: EDITOR & KHU VỰC HIỂN THỊ KẾT QUẢ

2.1 Editor là gì?

Vùng giữa màn hình = Editor — nơi hiển thị nội dung file và kết quả.

3 chế độ hiển thị quan trọng:

[Code View]     ← Xem code/text thuần (mặc định khi mở .md, .json)
[Preview]       ← Xem kết quả render (quan trọng với .html, .md)
[Split View]    ← Xem cả Code và Preview cùng lúc (recommend!)

Chuyển chế độ:
- Nhấn icon Preview (mắt 👁️) ở góc trên phải Editor
- Hoặc Click phải tab → Split Editor Right

2.2 Xem kết quả HTML (Vibe Coding)

Khi AI tạo file .html, bạn muốn xem ngay trên màn hình:

1. AI tạo xong file index.html
2. Click file → nó mở trong Editor
3. Click icon Preview (👁️) ở góc trên phải
4. Thấy webpage xuất hiện ngay bên phải!

Cách 2 — Mở trong trình duyệt thật:

1. Click phải file .html trong Explorer
2. Chọn "Reveal in Finder" (Mac) / "Open in Explorer" (Windows)
3. Double-click file → Mở trong Chrome/Edge

Cách 3 — AI mở browser tự động:

Gõ: "Hãy preview file vừa tạo trong browser"
→ AI Agent tự mở browser, chụp screenshot, gửi lại cho bạn xem

NGÀY 2: NGHỆ THUẬT NÓI CHUYỆN VỚI AI

🧠 Bài học hôm nay

Prompt = Câu lệnh bạn gửi cho AI. Prompt tốt không phải là câu dài nhất — mà là câu rõ ràng nhất.

Hãy nhớ công thức C.L.E.A.R:

C — Context    (Bối cảnh)  : Bạn là ai? Đang làm gì?
L — Length     (Độ dài)    : Output cần ngắn hay dài?
E — Example    (Ví dụ)     : Cho AI xem mẫu kết quả bạn muốn
A — Action     (Hành động) : Động từ cụ thể: Tạo / Phân tích / Viết / Tóm tắt
R — Result     (Kết quả)   : Định dạng output: bảng / PDF / web / email

So sánh prompt:

❌ Prompt yếu: "Viết email cho khách hàng"

✅ Prompt mạnh (C.L.E.A.R):

[Context]  Tôi là nhân viên CSKH, vừa nhận phàn nàn từ khách về giao hàng trễ.
[Action]   Viết email xin lỗi chuyên nghiệp.
[Example]  Tone: lịch sự, chân thành, không quá trang trọng.
[Length]   Không quá 150 từ.
[Result]   Tiếng Việt, định dạng email hoàn chỉnh (Kính gửi... / Kính thư...).

⚡ Thực hành ngay

Bài tập "Viết lại Prompt" — 20 phút:

Lấy 3 công việc bạn thực sự đang làm tuần này. Viết prompt C.L.E.A.R cho từng việc. Sau đó thử với AI và so sánh kết quả với prompt đơn giản ban đầu.

Ví dụ bộ 3 prompt theo ngành:

Kế toán:

[Context]  Tôi làm kế toán, có file Excel 200 dòng chi phí tháng 2.
[Action]   Phân tích và tóm tắt: tổng chi, top 5 hạng mục tốn nhiều nhất,
           so sánh với tháng 1 (tháng 1 tổng 450 triệu).
[Data]     [paste dữ liệu vào đây]
[Result]   Bảng tóm tắt + 3 nhận xét quan trọng + 1 khuyến nghị.

Nhân sự:

[Context]  Tôi là HR, cần thông báo chính sách nghỉ phép mới cho toàn công ty.
[Action]   Viết thông báo nội bộ về: tăng phép năm từ 12 lên 15 ngày,
           thêm 2 ngày nghỉ sức khoẻ tâm thần.
[Result]   Tiếng Việt, tone thân thiện nhưng chuyên nghiệp, kèm FAQ 5 câu hỏi thường gặp.

Kinh doanh:

[Context]  Tôi chuẩn bị gặp khách hàng tiềm năng là công ty sản xuất 500 nhân viên.
[Action]   Tạo checklist 10 câu hỏi để khám phá nhu cầu trong buổi gặp đầu tiên.
[Result]   Câu hỏi mở, sắp xếp theo logic: từ khái quát → cụ thể → vấn đề → giải pháp.

💡 Mẹo pro

Khi AI trả lời chưa đúng ý, đừng viết lại từ đầu. Chỉ cần nói: "Gần đúng rồi! Nhưng hãy [thêm/bỏ/thay đổi] phần [X]." AI sẽ chỉ sửa những gì bạn chỉ ra.

NGÀY 5: SIÊU NĂNG LỰC SỐ 2 — PHÂN TÍCH DỮ LIỆU KHÔNG CẦN EXCEL

🧠 Bài học hôm nay

Bạn có một file Excel 500 dòng. Bạn muốn biết: tổng theo tháng, top 10, xu hướng, bất thường. Với Excel truyền thống, bạn cần biết VLOOKUP, PIVOT TABLE, và mất 1-2 tiếng.

Với Antigravity: paste dữ liệu vào → mô tả muốn gì → nhận kết quả trong 2 phút.

Kỹ thuật "Data Dump & Ask":

Bước 1: Copy dữ liệu từ Excel (Ctrl+A → Ctrl+C)
Bước 2: Paste vào chat Antigravity
Bước 3: Mô tả câu hỏi phân tích
Bước 4: Nhận kết quả + biểu đồ

⚡ Thực hành ngay

Lấy bất kỳ file dữ liệu nào từ công việc (có thể xoá thông tin nhạy cảm trước), sau đó thử 3 câu hỏi phân tích:

Câu hỏi 1 — Tổng quan:

Đây là dữ liệu [loại dữ liệu] của tôi:
[PASTE DATA]

Hãy cho tôi biết:
- Tổng số bản ghi
- Khoảng thời gian dữ liệu bao phủ
- 3 insight nổi bật nhất
- Có dữ liệu bất thường hoặc lỗi không?

Câu hỏi 2 — So sánh:

Từ dữ liệu trên, hãy so sánh hiệu suất theo [tuần/tháng/quý/phòng ban].
Tạo bảng so sánh với % tăng giảm và màu sắc (xanh = tốt, đỏ = xấu).

Câu hỏi 3 — Trực quan hoá:

Tạo trang HTML với dashboard trực quan cho dữ liệu trên.
Gồm: 1 biểu đồ cột, 1 biểu đồ đường xu hướng, bảng top 5.
Màu sắc chuyên nghiệp, tiêu đề tiếng Việt.

💡 Mẹo pro

Nếu dữ liệu quá lớn để paste, hãy copy 50-100 dòng mẫu đại diện. AI có thể phân tích mẫu và bạn áp dụng logic tương tự.

Tôi cần: hiểu nguyên tắc mỗi phương pháp + cách áp dụng ngay + bảng so sánh.

## Quy trình
1. Xác nhận loại báo cáo (tuần/tháng/dự án/sự cố)

Mọi ứng dụng web đều được tạo từ HTML + CSS + JavaScript.

NGÀY 25 (MỚI): MCP — KẾT NỐI ANTIGRAVITY VỚI MỌI HỆ THỐNG

"Từ AI Biệt Lập Đến AI Kết Nối Thế Giới"

═══════════════════════════════════════════

Vị trí: Thay thế/bổ sung cho nội dung Ngày 25 (Automation Mindset)
Phần Automation Mindset giữ nguyên — MCP là phần thực hành nâng cao

🧠 Bài học hôm nay — MCP là gì?

MCP = Model Context Protocol — ngôn ngữ chung để Antigravity "nói chuyện" với các phần mềm khác.

Hãy nghĩ theo nghĩa đơn giản:

Không có MCP:
Antigravity ←→ Chỉ làm việc trong workspace của nó
→ Muốn dữ liệu Google Sheets? Copy-paste thủ công.
→ Muốn gửi email? Tự copy nội dung sang Gmail.
→ Muốn xem lịch họp? Phải mở riêng ra xem.

Có MCP:
Antigravity ←→ Kết nối trực tiếp với:
  ├── Google Drive / Sheets / Docs ← Đọc/ghi file real-time
  ├── Gmail / Outlook              ← Soạn và gửi email
  ├── Google Calendar              ← Xem lịch, đặt meeting
  ├── Slack / Teams                ← Gửi tin nhắn, tóm tắt kênh
  ├── Notion / Confluence          ← Viết/cập nhật tài liệu
  ├── Jira / Trello                ← Cập nhật task, tạo ticket
  └── Database nội bộ             ← Truy vấn dữ liệu công ty

Ví dụ thực tế sau khi có MCP:

"Hãy đọc Google Sheet báo cáo doanh số tuần này, so sánh với tuần trước, viết email tóm tắt và gửi cho team@company.com, sau đó đăng tóm tắt lên kênh #sales trên Slack."

← 1 câu lệnh. AI tự làm tất cả. Bạn không cần làm gì cả.

MCP trong thế giới thực — Không cần biết kỹ thuật

Cách hình dung: MCP như "ổ cắm điện đa năng" — Antigravity cắm vào bất kỳ thiết bị nào có MCP connector.

Bảng MCP servers phổ biến cho văn phòng:

MCP Server Kết nối với Bạn có thể nhờ AI làm
Google Drive MCP Google Drive Đọc, tạo, sửa file Docs/Sheets/Slides
Gmail MCP Gmail Đọc inbox, soạn và gửi email, label
Google Calendar MCP Google Calendar Xem lịch, tạo event, tìm thời gian rảnh
Slack MCP Slack Đọc kênh, gửi tin, tóm tắt conversation
Notion MCP Notion Đọc/viết pages, cập nhật database
GitHub MCP GitHub Xem code, tạo issue, review PR
Jira MCP Jira Tạo ticket, cập nhật status, xem sprint

Thiết lập MCP — Hướng dẫn cho non-tech

Bước 1: Hiểu cấu trúc

MCP server được cấu hình qua file settings.json:

{
  "mcpServers": {
    "google-drive": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-gdrive"],
      "env": {
        "GDRIVE_CREDENTIALS": "path/to/credentials.json"
      }
    },
    "gmail": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-gmail"],
      "env": {
        "GMAIL_OAUTH_TOKEN": "your-token-here"
      }
    }
  }
}

Giải thích đơn giản: Đây như "danh bạ điện thoại" — bạn đăng ký số của từng dịch vụ để Antigravity biết cách gọi đến.

Bước 2: Xác thực (Authentication)

Mỗi MCP server cần bạn cho phép Antigravity truy cập:

Google Drive MCP setup:
1. Antigravity yêu cầu: "Cần quyền truy cập Google Drive"
2. Mở link xác thực trong trình duyệt
3. Đăng nhập Google → Chấp nhận quyền
4. Copy token về dán vào Antigravity
→ Xong! Antigravity có thể đọc/ghi Drive của bạn

Giống như lần đầu đăng nhập app điện thoại bằng tài khoản Google.

Bước 3: Test kết nối

Sau khi setup, test ngay:

"Hãy liệt kê 5 file gần nhất trong Google Drive của tôi"
→ Nếu AI liệt kê được tên file thật → Kết nối thành công!

Use cases MCP mạnh nhất cho văn phòng
